# 03. 테스트 하네스 (Test Harness)

> AI 시스템의 동작을 자동으로 검증하기 위한 테스트 인프라 설계를 다룹니다.

---

## AI 테스트가 어려운 이유

일반 소프트웨어와 달리 AI 시스템은 다음과 같은 특성 때문에 테스트가 까다롭습니다.

```
일반 소프트웨어: 입력 A → 항상 출력 B  (결정적)
AI 시스템:       입력 A → 출력 B, C, D 중 하나  (확률적)
```

- **비결정성**: 같은 프롬프트에 다른 답변
- **의미론적 동등성**: "맞다"와 "네, 맞습니다"는 다르지만 같은 의미
- **포맷 민감성**: 출력 형식이 매번 조금씩 다를 수 있음
- **컨텍스트 의존성**: 이전 대화가 결과에 영향

---

## 테스트 하네스 핵심 패턴

### 1. 골든 파일 테스트 (Golden File Test)

**승인된 참조 출력**을 파일로 저장해두고, 새 응답과 비교합니다.

```python
# golden_test.py
import json
from pathlib import Path

def run_golden_test(prompt: str, golden_file: str, model_fn):
    golden_path = Path(f"tests/golden/{golden_file}")
    
    actual = model_fn(prompt)
    
    if golden_path.exists():
        expected = golden_path.read_text()
        assert actual == expected, f"출력이 변경되었습니다:\n{actual}"
    else:
        # 처음 실행 시 골든 파일 생성
        golden_path.write_text(actual)
        print(f"골든 파일 생성: {golden_file}")
```

**업데이트 방법**: 골든 파일을 의도적으로 갱신할 때는 리뷰어가 변경 내용을 승인해야 합니다.

---

### 2. 의미론적 유사도 테스트

정확히 일치하지 않아도 의미가 같으면 통과하는 방식.

```python
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer("all-MiniLM-L6-v2")

def semantic_assert(actual: str, expected: str, threshold: float = 0.85):
    emb_a = model.encode(actual, convert_to_tensor=True)
    emb_e = model.encode(expected, convert_to_tensor=True)
    score = util.cos_sim(emb_a, emb_e).item()
    assert score >= threshold, (
        f"의미 유사도 {score:.2f} < {threshold}\n"
        f"실제: {actual}\n기대: {expected}"
    )
```

### 3. 구조 검증 테스트

JSON 출력이나 특정 포맷을 요구할 때 스키마를 검증합니다.

```python
from pydantic import BaseModel
import json

class AnalysisResult(BaseModel):
    summary: str
    score: float  # 0.0 ~ 1.0
    tags: list[str]

def test_structured_output(prompt: str, model_fn):
    raw = model_fn(prompt)
    
    try:
        data = json.loads(raw)
        result = AnalysisResult(**data)
        assert 0.0 <= result.score <= 1.0
        assert len(result.tags) > 0
    except (json.JSONDecodeError, ValueError) as e:
        raise AssertionError(f"구조화된 출력 검증 실패: {e}\n응답: {raw}")
```

### 4. 프롬프트 회귀 테스트 스위트

프롬프트 변경 시 기존 케이스들이 여전히 통과하는지 검사합니다.

```python
# tests/test_prompts.py
import pytest

REGRESSION_CASES = [
    {
        "id": "greeting_ko",
        "prompt": "안녕하세요",
        "must_contain": ["안녕", "반갑"],
        "must_not_contain": ["Hello", "Hi"],
    },
    {
        "id": "json_format",
        "prompt": "사용자 정보를 JSON으로 반환해줘: 이름=홍길동, 나이=30",
        "schema": {"name": str, "age": int},
    },
]

@pytest.mark.parametrize("case", REGRESSION_CASES, ids=[c["id"] for c in REGRESSION_CASES])
def test_regression(case, llm_client):
    response = llm_client.complete(case["prompt"])

    if "must_contain" in case:
        for keyword in case["must_contain"]:
            assert keyword in response, f"'{keyword}' 누락"

    if "must_not_contain" in case:
        for keyword in case["must_not_contain"]:
            assert keyword not in response, f"'{keyword}' 포함됨"
```

## CI/CD 통합

```yaml
# .github/workflows/llm-tests.yml
name: LLM Regression Tests

on:
  pull_request:
    paths:
      - "prompts/**"
      - "src/ai/**"

jobs:
  regression:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run LLM tests
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          pytest tests/regression/ -v --tb=short
          pytest tests/integration/ -v --tb=short -m "not slow"
```

## 비용 관리 전략

LLM 테스트는 API 비용이 발생하므로 전략적으로 실행해야 합니다.

| 전략 | 방법 |
|------|------|
| **캐싱** | 동일 프롬프트 응답을 로컬에 캐시 |
| **모킹(mocking)** | 단위 테스트에서는 고정 응답 사용 |
| **선택적 실행** | 프롬프트 관련 파일 변경 시에만 LLM 테스트 실행 |
| **저렴한 모델 우선** | CI에서는 Haiku 등 저렴한 모델로 1차 검사 |

```python
# 응답 캐싱 예시
import hashlib, json
from pathlib import Path

CACHE_DIR = Path(".llm_cache")

def cached_complete(prompt: str, model_fn) -> str:
    key = hashlib.sha256(prompt.encode()).hexdigest()[:16]
    cache_file = CACHE_DIR / f"{key}.json"
    
    if cache_file.exists():
        return json.loads(cache_file.read_text())["response"]
    
    response = model_fn(prompt)
    CACHE_DIR.mkdir(exist_ok=True)
    cache_file.write_text(json.dumps({"prompt": prompt, "response": response}))
    return response
```
