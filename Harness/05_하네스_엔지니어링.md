# 05. 하네스 엔지니어링 실전

> AI 하네스를 설계하고 구현할 때 적용하는 원칙, 패턴, 실전 전략을 다룹니다.

---

## 핵심 설계 원칙

### 1. 최소 권한 원칙 (Least Privilege)

에이전트에게 꼭 필요한 권한만 부여합니다. 권한은 기본적으로 거부하고, 필요한 것만 명시적으로 허용합니다.

```json
// Bad: 너무 넓은 권한
{ "allow": ["Bash(*)", "Write", "WebFetch"] }

// Good: 필요한 것만 허용
{
  "allow": [
    "Read",
    "Edit(src/**)",
    "Bash(npm run build)",
    "Bash(npm run test)"
  ],
  "deny": ["Bash(rm *)", "WebFetch"]
}
```

### 2. 관찰 가능성 (Observability)

하네스 내 모든 도구 호출은 추적 가능해야 합니다.

```python
import logging
import time
from functools import wraps

logger = logging.getLogger("harness")

def observable(tool_fn):
    @wraps(tool_fn)
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        logger.info(f"[TOOL] {tool_fn.__name__} 시작: args={args[:2]}")
        
        try:
            result = tool_fn(*args, **kwargs)
            elapsed = time.perf_counter() - start
            logger.info(f"[TOOL] {tool_fn.__name__} 완료: {elapsed:.2f}s")
            return result
        except Exception as e:
            logger.error(f"[TOOL] {tool_fn.__name__} 실패: {e}")
            raise
    
    return wrapper

@observable
def edit_file(path: str, content: str):
    ...
```

### 3. 실패 격리 (Fault Isolation)

하나의 도구 실패가 전체 에이전트를 중단시키지 않아야 합니다.

```python
class SafeToolExecutor:
    def __init__(self, max_retries: int = 2, timeout: float = 30.0):
        self.max_retries = max_retries
        self.timeout = timeout

    def execute(self, tool_fn, *args, **kwargs):
        for attempt in range(self.max_retries + 1):
            try:
                return tool_fn(*args, **kwargs)
            except TimeoutError:
                if attempt == self.max_retries:
                    return ToolResult(error="타임아웃: 작업이 너무 오래 걸립니다")
            except PermissionError as e:
                return ToolResult(error=f"권한 거부: {e}")
            except Exception as e:
                if attempt == self.max_retries:
                    return ToolResult(error=f"실패: {e}")
```

### 4. 결정론적 경계 (Deterministic Boundaries)

AI는 비결정적이지만, 하네스 레이어는 결정론적이어야 합니다.

```
비결정적 영역 (AI)         결정론적 영역 (하네스)
──────────────────────    ──────────────────────────
- 어떤 도구를 쓸지         - 권한이 있는지 없는지
- 어떤 코드를 생성할지     - 파일이 존재하는지
- 어떤 답변을 할지         - 명령이 성공했는지 실패했는지
```

## 하네스 설계 패턴

### 패턴 1: 가드레일 레이어 (Guardrail Layer)

에이전트 출력이 정책을 위반하는지 검사합니다.

```python
class GuardrailLayer:
    BLOCKED_PATTERNS = [
        r"rm\s+-rf",
        r"DROP\s+TABLE",
        r"sudo\s+",
    ]

    def validate_bash(self, command: str) -> tuple[bool, str]:
        import re
        for pattern in self.BLOCKED_PATTERNS:
            if re.search(pattern, command, re.IGNORECASE):
                return False, f"차단된 패턴 감지: {pattern}"
        return True, ""

    def validate_file_write(self, path: str) -> tuple[bool, str]:
        protected = [".env", "secrets.json", "credentials.json"]
        if any(path.endswith(p) for p in protected):
            return False, f"보호된 파일 수정 불가: {path}"
        return True, ""
```

### 패턴 2: 샌드박스 실행 (Sandbox Execution)

위험한 명령을 격리된 환경에서 실행합니다.

```python
import subprocess

def sandboxed_bash(command: str, workdir: str) -> dict:
    result = subprocess.run(
        command,
        shell=True,
        cwd=workdir,
        capture_output=True,
        text=True,
        timeout=30,
        # 환경변수 최소화
        env={
            "PATH": "/usr/local/bin:/usr/bin:/bin",
            "HOME": workdir,
        }
    )
    return {
        "stdout": result.stdout,
        "stderr": result.stderr,
        "returncode": result.returncode,
    }
```

### 패턴 3: 비용 감시 (Cost Monitor)

토큰 사용량과 API 비용을 실시간 추적합니다.

```python
class CostMonitor:
    # claude-sonnet-4-6 기준 (2025)
    PRICE_PER_1K = {"input": 0.003, "output": 0.015}

    def __init__(self, budget_usd: float = 1.0):
        self.budget = budget_usd
        self.spent = 0.0
        self.input_tokens = 0
        self.output_tokens = 0

    def record(self, input_tokens: int, output_tokens: int):
        cost = (
            input_tokens / 1000 * self.PRICE_PER_1K["input"]
            + output_tokens / 1000 * self.PRICE_PER_1K["output"]
        )
        self.spent += cost
        self.input_tokens += input_tokens
        self.output_tokens += output_tokens

        if self.spent >= self.budget:
            raise BudgetExceeded(
                f"예산 초과: ${self.spent:.4f} / ${self.budget}"
            )

    def summary(self) -> dict:
        return {
            "input_tokens": self.input_tokens,
            "output_tokens": self.output_tokens,
            "total_cost_usd": round(self.spent, 4),
        }
```

### 패턴 4: 상태 스냅샷 (State Snapshot)

에이전트 실행 전후 상태를 저장해 복구 가능하게 합니다.

```python
import shutil
from datetime import datetime
from pathlib import Path

class StateSnapshot:
    def __init__(self, target_dir: str):
        self.target = Path(target_dir)
        self.snapshot_dir = Path(".harness_snapshots")

    def take(self, label: str = "") -> Path:
        ts = datetime.now().strftime("%Y%m%d_%H%M%S")
        name = f"{ts}_{label}" if label else ts
        dest = self.snapshot_dir / name
        shutil.copytree(self.target, dest)
        return dest

    def restore(self, snapshot_path: Path):
        if self.target.exists():
            shutil.rmtree(self.target)
        shutil.copytree(snapshot_path, self.target)

# 사용 예시
snapshot = StateSnapshot("src/")
backup = snapshot.take("before_refactor")
try:
    agent.run("전체 코드를 리팩토링해줘")
except Exception:
    snapshot.restore(backup)  # 실패 시 원상 복구
```

## 다중 에이전트 하네스

여러 에이전트가 협력할 때의 하네스 구조입니다.

```
┌─────────────────────────────────────────────────┐
│              오케스트레이터 하네스                │
│                                                 │
│  사용자 요청                                     │
│      ↓                                          │
│  [오케스트레이터 에이전트]                        │
│      ├─→ [탐색 에이전트] (Read-only 권한)         │
│      ├─→ [코드 작성 에이전트] (Edit 권한)         │
│      └─→ [검증 에이전트] (Bash 읽기 전용 권한)    │
│                                                 │
│  각 에이전트는 독립된 권한 프로파일을 가짐        │
└─────────────────────────────────────────────────┘
```

## 하네스 체크리스트

프로덕션 AI 시스템 배포 전 점검 항목:

```
[ ] 권한
    ☐ 최소 권한 원칙 적용
    ☐ 민감 파일/명령 차단 목록 정의
    ☐ 권한 에스컬레이션 불가 확인

[ ] 관찰 가능성
    ☐ 모든 도구 호출 로깅
    ☐ 비용 모니터링 설정
    ☐ 에러 알림 설정

[ ] 안전성
    ☐ 타임아웃 설정
    ☐ 무한 루프 방지 (max_iterations)
    ☐ 예산 한도 설정
    ☐ 롤백 메커니즘 구현

[ ] 테스트
    ☐ 골든 파일 테스트 작성
    ☐ 권한 거부 케이스 테스트
    ☐ 예산 초과 처리 테스트
```
