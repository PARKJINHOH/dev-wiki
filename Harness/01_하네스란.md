# 01. 하네스(Harness)란 무엇인가

> AI·소프트웨어 공학에서 하네스의 정의, 기원, 핵심 역할을 설명합니다.

---

## 원래 의미

"하네스(harness)"는 원래 말(horse)에 씌우는 **마구(馬具)** — 고삐, 안장, 줄 등의 장비 세트를 뜻합니다. 공학 용어로는 **복잡한 시스템을 제어하고 안전하게 활용하기 위한 래퍼 인프라**를 의미합니다.

소프트웨어에서 처음 등장한 것은 **테스트 하네스(Test Harness)** 개념으로, 단위 테스트나 통합 테스트를 자동화하기 위해 대상 코드를 감싸는 프레임워크를 말했습니다.

---

## AI에서의 하네스

AI·LLM 분야에서 하네스는 다음을 포함하는 넓은 개념입니다:

### 1. 평가 하네스 (Evaluation Harness)

모델의 성능을 표준화된 방식으로 측정하는 프레임워크.

```
사용자 요청 → [ 평가 하네스 ] → 모델 출력 → 점수/지표 계산
                    ↑
             벤치마크 데이터셋, 프롬프트 템플릿, 메트릭 정의
```

- **예시**: EleutherAI `lm-evaluation-harness`, OpenAI Evals, HELM
- **목적**: 다른 모델·버전 간 공정한 성능 비교

---

### 2. 테스트 하네스 (Test Harness)

AI 시스템이 기대 동작을 유지하는지 자동으로 검증하는 인프라.

```
테스트 케이스 → [ 테스트 하네스 ] → AI 응답 → 검증(assertion)
```

- **예시**: 골든 파일(golden file) 테스트, 프롬프트 회귀 테스트
- **목적**: 모델 교체·프롬프트 수정 시 회귀 탐지

---

### 3. 실행 하네스 (Execution Harness)

에이전트가 도구(tool)를 호출하고 외부 시스템과 상호작용할 때 **권한·안전성·관찰 가능성**을 강제하는 실행 환경.

```
에이전트 요청 → [ 실행 하네스 ] → 도구 실행 → 결과 반환
                    ↓
             권한 체크 / 훅 실행 / 로그 기록
```

- **예시**: Claude Code 하네스, LangChain AgentExecutor, AutoGen Runtime
- **목적**: 에이전트 행동을 통제하고 감사(audit) 가능하게 만듦

---

## 하네스의 공통 구성 요소

| 구성 요소 | 역할 |
|-----------|------|
| **입력 어댑터** | 다양한 포맷의 입력을 표준화 |
| **실행 엔진** | 모델/도구를 실제로 호출 |
| **훅(Hook)** | 실행 전·후에 끼어드는 콜백 |
| **관찰 가능성** | 로그, 메트릭, 트레이스 수집 |
| **권한 레이어** | 허용/거부 정책 강제 |
| **출력 어댑터** | 결과를 표준 포맷으로 변환 |

---

## 하네스가 없으면 생기는 문제

```
❌ 하네스 없음
   - 모델 응답이 어디서 왔는지 추적 불가
   - 권한 없는 도구 호출을 막을 수 없음
   - 프롬프트 변경 영향을 측정하기 어려움
   - 에이전트 루프가 무한히 실행될 수 있음

✅ 하네스 있음
   - 모든 도구 호출이 로그에 남음
   - 허용된 작업만 실행됨
   - A/B 테스트·회귀 감지가 가능
   - 타임아웃·비용 한도를 강제할 수 있음
```
