## 하네스란?

소프트웨어 공학에서 **하네스(Harness)**는 특정 컴포넌트를 안전하고 재현 가능하게 실행·검증하기 위한 **래퍼(wrapper) 인프라**를 의미합니다.

AI 분야에서는 다음 세 가지 맥락에서 주로 사용됩니다:

```
┌─────────────────────────────────────────────────┐
│                 AI 하네스 유형                   │
├─────────────────┬───────────────────────────────┤
│ 평가 하네스      │ 모델 성능을 벤치마크로 측정     │
│ 테스트 하네스    │ AI 시스템의 동작을 자동 검증    │
│ 실행 하네스      │ AI 에이전트의 도구·권한을 제어  │
└─────────────────┴───────────────────────────────┘
```

---

## 왜 중요한가

- LLM은 비결정적(non-deterministic)이어서 **재현 가능한 실행 환경**이 필수
- 에이전트가 외부 도구를 호출할 때 **권한·안전성 경계**를 강제해야 함
- 모델 업그레이드나 프롬프트 변경 시 **회귀(regression) 감지** 필요
