# 08. 에이전트 (Agents)

> 복잡한 작업을 처리하는 서브에이전트 시스템을 설명합니다.

---

## 에이전트란?

**에이전트(Agent)**는 특정 역할에 특화된 AI 서브에이전트입니다. Claude Code가 복잡한 작업을 처리할 때 메인 Claude가 모든 걸 처리하는 대신, 적합한 에이전트를 불러서 작업을 위임합니다.

```
사용자: "이 코드베이스에서 보안 취약점 전체 점검해줘"

메인 Claude:
  ├── 코드 탐색 에이전트 실행  → 전체 파일 구조 파악
  ├── 보안 리뷰 에이전트 실행  → 취약점 분석
  └── 결과 통합 후 보고
```

에이전트를 활용하면 메인 컨텍스트 창을 절약하면서 더 깊이 있는 작업이 가능합니다.

---

## 왜 에이전트가 필요한가?

| 단일 Claude | 에이전트 활용 |
|------------|-------------|
| 모든 작업을 순차적으로 처리 | 병렬로 여러 작업 동시 처리 |
| 컨텍스트 창 하나에 모든 내용 누적 | 작업별 별도 컨텍스트로 분리 |
| 큰 코드베이스 탐색 시 느림 | 탐색 전문 에이전트로 빠르게 처리 |
| 복잡한 멀티스텝 작업에 한계 | 단계별로 에이전트 분리해서 처리 |

---

## 기본 제공 에이전트

Claude Code에는 기본적으로 다음 에이전트들이 내장되어 있습니다.

### Explore (탐색 전문)
코드베이스 빠른 탐색에 특화된 에이전트입니다.

- 파일 패턴 검색
- 키워드로 코드 검색
- 프로젝트 구조 파악
- API 엔드포인트 추출 등

**사용 시점:** 파일을 찾거나 코드 구조를 파악해야 할 때

---

### Plan (아키텍처 설계)
구현 계획과 설계를 담당합니다.

- 구현 전략 수립
- 중요 파일 식별
- 아키텍처 트레이드오프 분석
- 단계별 실행 계획 작성

**사용 시점:** 큰 기능을 구현하기 전 설계가 필요할 때

---

### general-purpose (범용)
복잡한 연구, 코드 검색, 멀티스텝 작업을 처리합니다.

- 복잡한 질문 조사
- 여러 파일에 걸친 코드 검색
- 멀티스텝 실행 작업

**사용 시점:** 다른 에이전트로 명확하게 분류되지 않는 복잡한 작업

---

### claude-code-guide (Claude Code 전문)
Claude Code 자체에 대한 질문을 처리합니다.

- Claude Code 기능 안내
- 슬래시 명령어 사용법
- MCP 서버 설정
- Claude API 사용법

**사용 시점:** Claude Code 사용법이 궁금할 때

## 사용자 정의 에이전트

프로젝트에 맞는 커스텀 에이전트를 직접 만들 수 있습니다.

**위치:** `.claude/agents/` 폴더에 마크다운 파일로 작성

```
프로젝트루트/
└── .claude/
    └── agents/
        ├── api-reviewer.md    ← 커스텀 API 리뷰 에이전트
        └── db-optimizer.md    ← 커스텀 DB 최적화 에이전트
```

**에이전트 파일 형식 예시:**
```markdown
---
name: api-reviewer
description: REST API 설계 원칙과 보안 취약점을 전문으로 리뷰합니다
tools: Read, Grep, Glob
---

당신은 REST API 전문 리뷰어입니다.
다음 기준으로 코드를 검토하세요:
- HTTP 메서드가 올바르게 사용되었는가
- 인증/인가 처리가 올바른가
- 입력값 검증이 충분한가
- 에러 응답 형식이 일관적인가
```

**`description` 필드가 중요합니다.** Claude가 이 설명을 읽고 어떤 상황에서 이 에이전트를 불러야 할지 판단합니다.

---

## 에이전트 병렬 실행

Claude는 서로 독립적인 작업을 여러 에이전트로 동시에 처리할 수 있습니다.

```
예시: 리팩토링 전 현황 파악

메인 Claude가 동시에 시작:
  에이전트 A → 파일 구조 탐색 중...
  에이전트 B → 테스트 커버리지 분석 중...
  에이전트 C → 의존성 그래프 파악 중...

모두 완료 → 결과 통합 → 리팩토링 계획 수립
```

---

## 에이전트 활용 팁

**명시적으로 에이전트 요청하기:**
```
> 탐색 에이전트를 써서 이 프로젝트에서 인증 관련 코드를 모두 찾아줘
> 설계 에이전트로 새 결제 모듈 구조를 먼저 계획해줘
```

**멀티스텝 작업에 활용하기:**
```
> 전체 코드베이스를 보안 점검해줘. 
  먼저 탐색 에이전트로 구조 파악하고, 
  그 다음 각 모듈별로 보안 리뷰 에이전트를 써줘.
```

---

## 에이전트와 컨텍스트 관리

에이전트는 메인 컨텍스트와 분리된 별도 컨텍스트에서 실행됩니다.

- 에이전트가 수집한 정보는 요약되어 메인 컨텍스트로 반환됨
- 덕분에 메인 컨텍스트를 아낄 수 있음
- 큰 코드베이스를 탐색할 때 특히 유용
