# 03. 핵심 개념

> Claude Code를 제대로 사용하기 위해 반드시 이해해야 할 개념들을 설명합니다.

---

## 1. 토큰 (Token)

AI 모델이 텍스트를 처리하는 기본 단위입니다.

- 영어 기준 약 4자 = 1토큰
- 한국어는 영어보다 토큰 소비가 많음 (약 1~2자 = 1토큰)
- 코드도 토큰으로 계산됨

**왜 중요한가?**
토큰을 사용할수록 비용이 발생합니다. 또한 한 번에 처리할 수 있는 토큰 수(컨텍스트 창)에 한계가 있습니다.

| 모델 | 최대 컨텍스트 | 특징 |
|------|-------------|------|
| claude-opus-4-7 | 200K 토큰 | 가장 강력, 비용 높음 |
| claude-sonnet-4-6 | 200K 토큰 | 균형 잡힌 기본값 |
| claude-haiku-4-5 | 200K 토큰 | 빠르고 저렴 |

---

## 2. 컨텍스트 창 (Context Window)

Claude가 한 번의 세션에서 기억할 수 있는 내용의 한계입니다.

```
[대화 시작]
  ├── 이전 대화 내용
  ├── 읽어들인 파일 내용
  ├── 실행 결과
  └── 현재 질문
        ↓
  모두 합쳐서 200K 토큰 안에 들어와야 함
```

**컨텍스트가 꽉 차면 어떻게 되나?**

- 오래된 내용부터 자동으로 잘려나감
- Claude가 앞서 논의한 내용을 기억하지 못할 수 있음
- `/compact` 명령어로 수동 압축하거나 `/clear`로 초기화 가능

**현재 사용량 확인:**
```bash
/context
```

## 3. 세션 (Session)

Claude Code를 실행하고 종료하는 한 단위입니다.

```
claude 실행 → [세션 시작]
  대화 진행...
  작업 수행...
claude 종료 → [세션 종료]
```

- 세션 내 대화는 컨텍스트 창에 쌓임
- 세션이 끝나면 대화 내용은 사라짐
- 단, **메모리 시스템**을 통해 중요한 정보는 다음 세션으로 이어갈 수 있음

**이전 세션 이어서 작업:**
```bash
claude --continue    # 마지막 세션 바로 재개
claude --resume      # 세션 목록에서 선택해서 재개
```

## 4. CLAUDE.md

Claude가 프로젝트를 이해하는 기준 문서입니다. 프로젝트 루트에 위치합니다.

```
my-project/
├── CLAUDE.md        ← 이 파일
├── src/
├── package.json
└── ...
```

**무엇을 담는가?**
- 프로젝트 개요와 기술 스택
- 코딩 컨벤션과 규칙
- 자주 쓰는 명령어
- 주의사항이나 금기 사항

**예시:**
```markdown
# My Project

## 기술 스택
- React 18, TypeScript, Vite
- API: Express, PostgreSQL

## 규칙
- 함수 이름은 camelCase
- 컴포넌트는 PascalCase
- 테스트 없이 PR 올리지 않기

## 자주 쓰는 명령어
- npm run dev: 개발 서버 실행
- npm test: 테스트 실행
```

Claude는 세션을 시작할 때 이 파일을 자동으로 읽습니다. 잘 작성할수록 Claude가 더 정확하게 일합니다.

**자동 생성:**
```bash
/init    # Claude가 프로젝트를 분석해서 CLAUDE.md 자동 생성
```

> 자세한 내용은 [05_CLAUDE_md_가이드.md](./05_CLAUDE_md_가이드.md)를 참고하세요.

## 5. 도구 (Tools)

Claude가 실제 작업을 수행할 때 사용하는 수단입니다. 사용자는 직접 도구를 호출하지 않고, Claude가 필요에 따라 자동으로 선택해 사용합니다.

| 도구 | 하는 일 |
|------|---------|
| Read | 파일 읽기 |
| Edit | 파일 일부 수정 |
| Write | 파일 전체 쓰기 |
| Glob | 파일 패턴으로 검색 |
| Grep | 파일 내용 검색 |
| Bash | 터미널 명령 실행 |
| Agent | 서브에이전트 실행 |
| WebSearch | 웹 검색 |
| WebFetch | URL 내용 가져오기 |

**도구 실행 시 승인 요청:**

Claude는 파일을 수정하거나 명령어를 실행하기 전에 사용자에게 확인을 요청합니다. 위험한 작업일수록 더 명시적으로 확인합니다.

```
Claude: src/utils.ts 파일의 12번째 줄을 수정하겠습니다. 진행할까요?
사용자: y (또는 n)
```

## 6. 허용 모드 (Permission Mode)

도구 실행 시 허락을 받는 방식입니다.

| 모드 | 설명 |
|------|------|
| default | 모든 도구 실행 전 허락 요청 |
| auto-approve | 안전한 도구는 자동 승인 |
| strict | 모든 작업에 명시적 허락 필요 |

> 설정 방법은 [09_설정_권한.md](./09_설정_권한.md)를 참고하세요.

## 7. 모델 (Model)

Claude Code는 세 가지 모델 중 하나를 사용합니다.

| 모델 | 속도 | 비용 | 적합한 작업 |
|------|------|------|------------|
| claude-opus-4-7 | 느림 | 높음 | 복잡한 아키텍처 설계, 어려운 버그 |
| claude-sonnet-4-6 | 보통 | 보통 | 대부분의 개발 작업 (기본값) |
| claude-haiku-4-5 | 빠름 | 낮음 | 단순 질문, 빠른 수정 |

**모델 변경:**
```bash
/model                         # 현재 모델 확인
/model claude-opus-4-7         # Opus로 변경
/model claude-haiku-4-5        # Haiku로 변경
```

## 8. 메모리 (Memory)

세션이 끝나도 사라지지 않는 영구 기억 공간입니다.

- 파일 형태로 로컬에 저장됨 (`~/.claude/projects/.../memory/`)
- 다음 세션 시작 시 자동으로 불러옴
- "기억해줘"라고 말하면 Claude가 적절한 항목을 저장함

**저장되는 내용:**
- 사용자의 역할, 선호도, 기술 수준
- "이런 방식으로 코드 작성해줘" 같은 피드백
- 프로젝트 진행 상황 및 결정 사항
- 외부 시스템 위치 (Jira 프로젝트 ID, 대시보드 URL 등)

> 자세한 내용은 [06_메모리_시스템.md](./06_메모리_시스템.md)를 참고하세요.

## 핵심 개념 요약

```
Claude Code 세션
│
├── 컨텍스트 창 (200K 토큰)
│   ├── CLAUDE.md 내용 (자동 로드)
│   ├── 메모리 인덱스 (자동 로드)
│   ├── 대화 내용 누적
│   └── 파일 내용 (읽을 때마다 추가)
│
├── 도구 → 실제 파일 수정, 명령 실행
│
└── 세션 종료 → 컨텍스트 사라짐
              → 메모리는 파일에 유지됨
```
