# 06. 메모리 시스템

> 세션이 끝나도 사라지지 않는 Claude의 영구 기억 시스템을 설명합니다.

---

## 메모리란?

기본적으로 Claude는 세션이 끝나면 대화 내용을 기억하지 못합니다. 하지만 **메모리 시스템**을 통해 중요한 정보를 파일로 저장해두면 다음 세션에서도 자동으로 불러옵니다.

```
세션 A: "나는 백엔드 개발자야. TypeScript 위주로 써줘"
       → 메모리에 저장됨

세션 B (다음 날): 새 세션 시작
       → 메모리 자동 로드
       → Claude가 이미 알고 있음: "TypeScript 위주로 작성해야 함"
```

---

## 저장 위치

메모리는 로컬 파일로 저장됩니다.

```
~/.claude/projects/
└── D--workspace-myProject/    ← 프로젝트별 폴더
    └── memory/
        ├── MEMORY.md          ← 메모리 인덱스 (목록)
        ├── user_role.md       ← 사용자 정보
        ├── feedback_style.md  ← 작업 방식 피드백
        └── project_status.md  ← 프로젝트 진행 상황
```

---

## 메모리 유형 4가지

### user — 사용자 프로필

사용자의 역할, 기술 수준, 선호도 등을 저장합니다.

```
저장 예시:
- "백엔드 개발자, Go 10년 경력, React는 처음"
- "코드 설명 시 Go와 비교해서 설명해달라고 함"
- "간결한 답변 선호, 긴 설명 불필요"
```

**효과:** Claude가 설명 방식이나 코드 스타일을 자동으로 맞춤

---

### feedback — 작업 방식 지침

Claude에게 "이렇게 일해줘" 또는 "이건 하지 마"를 저장합니다.

```
저장 예시:
- "테스트에서 DB 목킹 금지. 실제 DB 연결로만 테스트"
- "응답 끝에 요약 추가하지 말 것"
- "한 PR에 여러 변경사항 묶는 방식 선호"
```

**효과:** Claude가 같은 피드백을 반복해서 받지 않아도 됨

---

### project — 프로젝트 진행 상황

현재 진행 중인 작업, 결정 사항, 배경 등을 저장합니다.

```
저장 예시:
- "인증 미들웨어 교체 작업 진행 중. 법무팀 컴플라이언스 요건 때문"
- "2026-05-01까지 모바일 릴리즈 예정. 이후 비중요 PR 동결"
- "결제 모듈은 외부 팀 담당, 직접 수정 금지"
```

**효과:** Claude가 프로젝트 맥락과 제약사항을 이해하고 더 적절한 제안을 함

---

### reference — 외부 시스템 위치

Jira, Grafana, Linear 등 외부 시스템의 위치 정보를 저장합니다.

```
저장 예시:
- "버그는 Linear 프로젝트 'BACKEND'에서 트래킹"
- "API 레이턴시 대시보드: grafana.internal/d/api-latency"
- "배포 파이프라인: github.com/company/infra/actions"
```

**효과:** Claude가 적절한 시스템을 참조해서 더 정확한 정보 제공

## 메모리 저장 방법

### 방법 1: 대화 중 자연어로 요청

```
> 나는 데이터 사이언티스트야. 기억해줘.
> 테스트 코드에서 mock 사용하지 말아줘. 기억해줘.
> 지금 인증 리팩토링 작업 중이야. 저장해줘.
```

Claude가 적절한 유형의 메모리 파일에 자동으로 저장합니다.

### 방법 2: /memory 명령어로 직접 관리

```bash
/memory    # 저장된 메모리 목록 확인 및 직접 편집
```

---

## 메모리 확인 및 관리

```bash
/memory    # 메모리 목록 열기
```

목록에서 개별 파일을 선택해서 내용 확인, 수정, 삭제를 할 수 있습니다.

**메모리 삭제를 요청하려면:**
```
> 내가 TypeScript 선호한다고 기억한 거 잊어줘.
```

---

## 메모리와 CLAUDE.md의 차이

| 항목 | 메모리 | CLAUDE.md |
|------|--------|-----------|
| 대상 | 사용자 개인 정보 / 피드백 | 프로젝트 공통 정보 |
| 저장 위치 | 로컬 홈 디렉토리 | 프로젝트 루트 |
| 작성 방식 | Claude가 대화 중 자동 저장 | 개발자가 직접 작성 |
| 팀 공유 | 불가 (개인 로컬) | 가능 (Git으로 공유) |
| 내용 예시 | "mock 쓰지 마" | "테스트 정책: Vitest 사용" |

---

## 메모리에 저장하면 안 되는 것

메모리는 세션을 넘어 유지되어야 할 정보에만 씁니다. 다음은 메모리 대신 다른 수단을 쓰는 게 적합합니다.

| 저장하면 안 되는 것 | 대신 쓸 것 |
|--------------------|-----------|
| 파일 경로나 코드 패턴 | 코드 자체 또는 CLAUDE.md |
| 최근 Git 변경 사항 | `git log` |
| 임시 작업 상태 | 현재 대화 컨텍스트 |
| 이미 CLAUDE.md에 있는 내용 | 중복 저장 불필요 |

## 활용 예시

**시나리오:** 새 팀원이 합류해서 처음 Claude Code를 쓰는 경우

```
1일차:
> 나는 프론트엔드 개발자야. React에는 익숙하지만 이 프로젝트 백엔드는 처음이야.
→ [user 메모리 저장]

> 답변 끝에 요약 붙이지 마. 짧게 말해줘.
→ [feedback 메모리 저장]

14일차:
→ 새 세션 시작해도 Claude는 이 사람이 프론트 개발자이고 짧게 답해야 한다는 걸 알고 있음
```
