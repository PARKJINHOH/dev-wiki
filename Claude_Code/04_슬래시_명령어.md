# 04. 슬래시 명령어

> Claude Code 대화 중 `/`로 시작하는 특수 명령어 전체 목록과 사용법입니다.

---

## 명령어란?

슬래시 명령어는 대화 중 Claude에게 말을 거는 게 아니라, **Claude Code 시스템 자체를 제어**하는 명령어입니다.

```
> 이 파일 분석해줘        ← Claude에게 하는 말 (자연어 요청)
> /compact               ← Claude Code 시스템 명령어 (슬래시 명령어)
```

---

## 세션 관리

### /init
프로젝트를 Claude Code와 함께 사용할 수 있도록 초기화합니다.

- 현재 디렉토리를 분석해서 `CLAUDE.md` 파일을 자동 생성
- 프로젝트 구조, 기술 스택, 주요 파일 경로 등을 기록

```bash
/init
```

**언제 사용하나:** 새 프로젝트에 Claude Code를 처음 도입할 때

---

### /status
현재 세션의 상태를 요약해서 보여줍니다.

- 사용 중인 모델명
- 누적 토큰 사용량
- 컨텍스트 남은 용량 (%)
- 현재 작업 디렉토리

```bash
/status
```

---

### /model
현재 사용 중인 AI 모델을 확인하거나 변경합니다.

```bash
/model                        # 현재 모델 확인
/model claude-opus-4-7        # Opus로 변경 (강력, 비용 높음)
/model claude-sonnet-4-6      # Sonnet으로 변경 (기본값)
/model claude-haiku-4-5       # Haiku로 변경 (빠름, 저렴)
```

**팁:** 복잡한 리팩토링은 Opus, 단순 수정이나 질문은 Haiku로 전환하면 비용을 아낄 수 있습니다.

---

### /resume
이전 대화 세션을 불러와 이어서 작업합니다.

```bash
/resume                  # 세션 목록에서 선택
claude --continue        # 실행 시 마지막 세션 바로 재개
claude --resume          # 실행 시 세션 선택해서 재개
```

**언제 사용하나:** 작업 중간에 터미널을 닫았거나 다음 날 이어서 할 때

---

### /clear
현재 대화 컨텍스트를 완전히 초기화합니다. 토큰 사용량도 0으로 리셋됩니다.

```bash
/clear
```

**언제 사용하나:** 완전히 다른 주제의 새 작업을 시작할 때

---

### /cost
현재 세션에서 사용된 토큰 비용을 확인합니다.

- 입력/출력 토큰 수 및 비용
- 캐시 적중률 (비용 절감 효과)
- 세션 총 비용

```bash
/cost
```

---

## 컨텍스트 관리

### /context
현재 컨텍스트 창의 사용 현황을 보여줍니다.

```bash
/context
```

**컨텍스트가 70% 이상 차면 `/compact` 사용을 권장합니다.**

---

### /compact
대화 내용을 요약(압축)해서 컨텍스트 공간을 확보합니다.

- 지금까지 대화를 핵심 위주로 요약
- 불필요한 반복 제거
- 압축 방식을 직접 지시할 수 있음

```bash
/compact                               # 기본 압축
/compact 코드 변경사항 위주로 요약해줘    # 특정 내용 중심
/compact 현재 버그 수정 진행상황만 남겨줘  # 작업 진행상황 중심
```

**언제 사용하나:** 작업이 길어져 컨텍스트가 70% 이상 찼을 때

---

### /add-dir
현재 컨텍스트에 추가 디렉토리를 포함합니다.

```bash
/add-dir ../shared-utils    # 상위 폴더의 다른 패키지
/add-dir ./packages/core    # 하위 패키지
```

**기본적으로 Claude는 현재 작업 디렉토리만 인식합니다.** 다른 폴더 코드를 참조해야 할 때 사용하세요.

---

### /memory
세션을 넘어 영구적으로 유지되는 Claude 메모리를 관리합니다.

```bash
/memory    # 저장된 메모리 목록 확인 및 관리
```

> 자세한 내용은 [06_메모리_시스템.md](./06_메모리_시스템.md)를 참고하세요.

---

## 설정 및 환경

### /config
Claude Code의 동작 설정을 확인하거나 변경합니다.

```bash
/config                            # 현재 설정 목록
/config set output.language ko    # 응답 언어 설정
```

---

### /doctor
Claude Code 실행 환경을 점검하고 문제를 진단합니다.

- API 키 설정 및 유효성
- Node.js 버전 호환성
- 필수 의존성 설치 상태
- 네트워크 연결 상태

```bash
/doctor
```

**Claude Code가 이상하게 동작할 때 가장 먼저 실행하세요.**

---

### /permissions
Claude Code가 할 수 있는 작업의 권한을 관리합니다.

```bash
/permissions                       # 현재 권한 목록
/permissions allow bash            # 터미널 명령어 허용
/permissions deny file.delete      # 파일 삭제 거부
```

> 자세한 내용은 [09_설정_권한.md](./09_설정_권한.md)를 참고하세요.

---

### /mcp
MCP(Model Context Protocol) 서버를 관리합니다.

```bash
/mcp                    # 연결된 MCP 서버 목록
/mcp add github         # GitHub MCP 서버 추가
```

> 자세한 내용은 [07_MCP_서버.md](./07_MCP_서버.md)를 참고하세요.

---

### /agents
사용 가능한 서브에이전트 목록을 확인하고 관리합니다.

```bash
/agents    # 에이전트 목록 확인
```

> 자세한 내용은 [08_에이전트.md](./08_에이전트.md)를 참고하세요.

---

## 작업 및 출력

### /export
현재 세션 내용을 파일로 내보냅니다.

```bash
/export                        # 기본 내보내기
/export --format markdown      # 마크다운 형식
```

---

### /review
현재 브랜치의 변경사항을 코드 리뷰합니다.

- Git diff 분석
- 버그 가능성, 코드 품질, 보안 이슈 리포트

```bash
/review                        # 전체 변경사항 리뷰
/review src/api/users.ts       # 특정 파일만 리뷰
```

**PR 올리기 전에 실행하면 놓친 문제를 미리 발견할 수 있습니다.**

---

### /vim
입력 모드를 Vim 키바인딩으로 전환합니다.

```bash
/vim        # Vim 모드 활성화
/vim off    # 비활성화
```

---

### /help
사용 가능한 명령어 목록과 사용법을 출력합니다.

```bash
/help             # 전체 명령어 목록
/help compact     # /compact 상세 설명
```

---

### /bug
Claude Code 자체 버그를 Anthropic에 보고합니다.

```bash
/bug
```
