# 01. Claude Code란?

> Claude Code가 무엇인지, 어떻게 동작하는지, 왜 쓰는지를 설명합니다.

---

## Claude Code란 무엇인가

**Claude Code**는 Anthropic이 만든 AI 기반 코딩 어시스턴트입니다. 터미널(명령줄)에서 실행되며, 개발자와 대화하면서 실제 코드 작업을 수행합니다.

ChatGPT 같은 일반 AI 챗봇과 가장 크게 다른 점은 이것입니다.

| 일반 AI 챗봇 | Claude Code |
|-------------|-------------|
| 대화창에서 텍스트로 소통 | 터미널에서 직접 실행 |
| 코드를 보여주는 것에 그침 | 파일을 직접 읽고 수정 |
| 명령어 실행 불가 | 터미널 명령어 직접 실행 |
| 프로젝트 구조 모름 | 전체 코드베이스 탐색 가능 |
| Git 연동 없음 | Git 커밋, PR 생성 등 가능 |

---

## 어떻게 동작하는가

Claude Code는 Claude AI 모델(Sonnet, Opus, Haiku)을 엔진으로 사용합니다. 사용자가 터미널에서 말을 걸면, Claude는 다양한 **도구(Tool)**를 사용해 실제 작업을 수행합니다.

```
사용자: "users.ts 파일에서 비밀번호 검증 로직에 버그가 있어"

Claude의 내부 동작:
  1. users.ts 파일 읽기 (Read 도구)
  2. 관련 파일 검색 (Grep 도구)
  3. 문제 파악 후 수정 (Edit 도구)
  4. 테스트 실행 (Bash 도구)
  5. 결과 보고
```

사용자에게는 Claude가 "어떤 파일을 읽고, 어떤 변경을 할지" 보여주고 허락을 받은 뒤 실행합니다.

---

## Claude Code가 할 수 있는 일

### 코드 작업
- 새 기능 구현
- 버그 수정
- 코드 리팩토링
- 코드 설명 및 문서화
- 보안 취약점 분석

### 프로젝트 탐색
- 파일 검색 및 내용 파악
- 코드 구조 분석
- 의존성 파악

### 개발 워크플로우
- 터미널 명령어 실행 (빌드, 테스트, 배포 등)
- Git 작업 (status, add, commit, push, PR 생성)
- 패키지 설치 및 환경 설정

### 외부 연동 (MCP)
- GitHub 이슈/PR 조회
- Jira, Slack 등 외부 서비스 연동
- 데이터베이스 쿼리

---

## Claude Code를 구성하는 요소

```
Claude Code
├── CLI 도구          → 터미널에서 실행하는 프로그램
├── AI 모델           → 실제 판단과 응답을 담당 (Sonnet, Opus, Haiku)
├── 도구(Tools)       → 파일 읽기, 명령 실행 등 실제 행동 수단
├── CLAUDE.md         → 프로젝트 맥락을 Claude에게 전달하는 문서
├── 메모리 시스템      → 세션을 넘어 기억을 유지하는 파일 기반 저장소
├── 설정 파일          → 권한, 환경변수, 자동화 등을 제어
└── MCP 서버          → 외부 도구와 연결하는 확장 시스템
```

---

## Claude Code vs Claude.ai (웹)

같은 Claude AI를 사용하지만 목적이 다릅니다.

| | Claude Code (CLI) | Claude.ai (웹) |
|--|------------------|----------------|
| 실행 환경 | 터미널 | 브라우저 |
| 파일 접근 | 직접 가능 | 수동 업로드만 |
| 코드 수정 | 직접 수행 | 텍스트로 제안만 |
| 명령 실행 | 직접 가능 | 불가 |
| Git 연동 | 가능 | 불가 |
| 적합한 용도 | 개발 작업 전반 | 일반 질문, 문서 작성 |

---

## 왜 쓰는가

### 속도
파일을 직접 열고 수정하므로 코드를 복사-붙여넣기하는 과정이 없습니다. "이 함수 고쳐줘"라고 하면 바로 수정됩니다.

### 맥락 이해
전체 코드베이스를 탐색할 수 있어서 "이 버그가 다른 파일에 미치는 영향"까지 파악합니다.

### 반복 작업 자동화
테스트 실행, 빌드, 린트 등을 자동으로 처리합니다.

### 학습 도구
코드를 설명하고, 왜 이렇게 구현했는지, 더 나은 방법은 무엇인지 가르쳐줍니다.
