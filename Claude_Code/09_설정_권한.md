# 09. 설정 & 권한

> Claude Code의 동작을 제어하는 설정 파일, 권한 시스템, 훅(Hook)을 설명합니다.

---

## 설정 파일 구조

Claude Code는 두 가지 레벨의 설정 파일을 사용합니다.

```
전역 설정:  ~/.claude/settings.json          ← 모든 프로젝트에 적용
프로젝트:   프로젝트루트/.claude/settings.json  ← 해당 프로젝트에만 적용
```

프로젝트 설정이 전역 설정보다 우선합니다. 팀 공유가 필요한 설정은 프로젝트 설정에, 개인 설정은 전역에 두세요.

---

## 설정 파일 항목

`settings.json`의 주요 항목입니다.

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git *)",
      "Read(**)",
      "Edit(**)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)"
    ]
  },
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_..."
      }
    }
  },
  "hooks": {
    "PreToolUse": [...],
    "PostToolUse": [...]
  },
  "env": {
    "NODE_ENV": "development"
  }
}
```

## 권한(Permissions) 시스템

Claude가 실행하는 도구에 대해 허용/거부를 명시적으로 설정합니다.

### 권한 확인

```bash
/permissions    # 현재 권한 목록 확인
```

### 권한 형식

```
도구이름(패턴)
```

**예시:**
```
Bash(npm run *)       → npm run으로 시작하는 모든 명령 허용
Bash(git *)           → git 명령어 허용
Read(**)              → 모든 파일 읽기 허용
Edit(src/**)          → src 폴더 내 파일 수정 허용
Bash(rm -rf *)        → 재귀 삭제 명령 거부
```

### 권한 설정 방법

**방법 1: 대화 중 허용**

Claude가 허락을 요청할 때 "항상 허용"을 선택하면 자동으로 `settings.json`에 추가됩니다.

**방법 2: 명령어로 설정**

```bash
/permissions allow "Bash(npm run *)"
/permissions deny "Bash(rm -rf *)"
```

**방법 3: settings.json 직접 편집**

```json
{
  "permissions": {
    "allow": ["Bash(npm run *)", "Bash(git *)"],
    "deny": ["Bash(sudo *)"]
  }
}
```

## 훅(Hook) 시스템

특정 이벤트 발생 시 자동으로 실행할 셸 명령어를 등록합니다.

**훅은 Claude가 아니라 시스템이 실행합니다.** Claude에게 "이럴 때 이렇게 해줘"라고 기억시키는 게 아니라, 실제로 자동 실행되는 트리거입니다.

### 훅 이벤트 종류

| 이벤트 | 발생 시점 |
|--------|---------|
| `PreToolUse` | Claude가 도구를 사용하기 직전 |
| `PostToolUse` | Claude가 도구를 사용한 직후 |
| `Notification` | Claude가 알림을 보낼 때 |
| `Stop` | Claude가 응답을 완료했을 때 |

### 훅 설정 예시

**편집할 때마다 린트 자동 실행:**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint --fix"
          }
        ]
      }
    ]
  }
}
```

**Claude가 완료할 때 macOS 알림 보내기:**
```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude가 작업을 완료했습니다\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
```

**특정 파일 수정 시 테스트 자동 실행:**
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit(src/**/*.ts)",
        "hooks": [
          {
            "type": "command",
            "command": "npm test -- --run"
          }
        ]
      }
    ]
  }
}
```

## 환경변수 설정

Claude Code 세션에서 사용할 환경변수를 설정 파일에 등록할 수 있습니다.

```json
{
  "env": {
    "NODE_ENV": "development",
    "API_BASE_URL": "http://localhost:4000"
  }
}
```

## /config 명령어

설정을 대화형으로 변경합니다.

```bash
/config                            # 현재 설정 확인
/config set output.language ko    # 응답 언어 한국어로
```

## 권장 설정 예시

### 일반 웹 개발 프로젝트

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git status)",
      "Bash(git diff *)",
      "Bash(git add *)",
      "Bash(git commit *)",
      "Bash(git push)",
      "Read(**)",
      "Edit(src/**)",
      "Edit(tests/**)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(sudo *)",
      "Bash(curl * | bash *)"
    ]
  }
}
```

### 엄격한 보안이 필요한 환경

```json
{
  "permissions": {
    "allow": [
      "Read(src/**)",
      "Bash(npm test)"
    ],
    "deny": [
      "Bash(*)",
      "Edit(**)"
    ]
  }
}
```

→ 파일 읽기와 테스트 실행만 허용, 파일 수정이나 기타 명령은 모두 수동 승인 필요

## 팀 설정 공유

프로젝트 설정은 Git으로 공유할 수 있습니다.

```
.claude/settings.json    → Git에 커밋 (팀 공유 설정)
```

단, API 토큰 등 민감한 정보는 `.gitignore`에 추가하거나 환경변수로 분리하세요.

```gitignore
# .gitignore에 추가
.claude/settings.local.json    ← 개인 설정 (Git 제외)
```

`settings.local.json`에는 개인 API 키나 로컬 전용 설정을 저장하고, `settings.json`에는 팀 공통 설정만 저장하세요.
