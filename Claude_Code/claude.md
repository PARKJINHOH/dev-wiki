## 세션 관리

### 📌 /init

- **역할**: 프로젝트를 Claude Code와 함께 사용할 수 있도록 초기화
- **동작**:
  - 현재 디렉토리를 분석하여 `CLAUDE.md` 파일 자동 생성
  - 프로젝트 구조, 기술 스택, 주요 파일 경로 등을 `CLAUDE.md`에 기록
- **언제 사용하나**:
  - 새 프로젝트를 시작할 때
  - 팀에 Claude Code를 처음 도입할 때
  - 기존 프로젝트에 Claude Code 설정을 추가할 때

```bash
/init
```

> `CLAUDE.md`는 Claude가 프로젝트를 이해하는 기준 문서입니다. 직접 편집해서 프로젝트 규칙이나 컨벤션을 추가하면 Claude가 더 정확하게 작업합니다.

---

### 📌 /status

- **역할**: 현재 세션의 상태를 요약해서 보여줌
- **확인 가능한 정보**:
  - 현재 사용 중인 AI 모델명
  - 누적 토큰 사용량 (입력 / 출력)
  - 컨텍스트 남은 용량 (%)
  - 현재 작업 디렉토리

```bash
/status
```

> 작업이 길어질수록 컨텍스트가 줄어드므로, 주기적으로 확인하는 습관을 들이세요.

---

### 📌 /model

- **역할**: 현재 사용 중인 AI 모델을 확인하거나 변경
- **사용 가능한 모델** (2025년 기준):
  - `claude-opus-4-7` — 가장 강력, 복잡한 작업에 적합
  - `claude-sonnet-4-6` — 속도와 성능의 균형 (기본값)
  - `claude-haiku-4-5` — 가장 빠름, 단순 작업에 적합

```bash
/model                        # 현재 모델 확인
/model claude-opus-4-7        # 모델 변경
```

> 복잡한 리팩토링이나 아키텍처 설계에는 Opus, 빠른 코드 수정이나 질문에는 Haiku를 활용하면 비용을 절약할 수 있습니다.

---

### 📌 /resume

- **역할**: 이전 대화 세션을 불러와 작업을 이어서 진행
- **동작**:
  - 마지막으로 작업하던 대화 컨텍스트를 복원
  - 진행 중이던 코드 수정, 논의 흐름 등을 그대로 이어받음

```bash
/resume                  # 슬래시 명령어로 재개
claude --continue        # CLI 실행 시 자동으로 이전 세션 이어서 시작
claude --resume          # 특정 세션 선택하여 재개
```

> 작업 중간에 터미널을 닫았거나, 다음 날 이어서 작업할 때 유용합니다.

---

### 📌 /clear

- **역할**: 현재 대화 컨텍스트를 완전히 초기화
- **동작**:
  - 지금까지의 대화 내용을 모두 지우고 새 세션처럼 시작
  - 토큰 사용량 초기화

```bash
/clear
```

> 주제가 완전히 바뀌는 새 작업을 시작할 때 사용하세요. 이전 대화의 잡음 없이 깔끔하게 시작할 수 있습니다.

---

### 📌 /cost

- **역할**: 현재 세션에서 사용된 토큰 비용을 확인
- **확인 가능한 정보**:
  - 입력 토큰 수 및 비용
  - 출력 토큰 수 및 비용
  - 캐시 적중률 (비용 절감 효과)
  - 세션 총 비용

```bash
/cost
```

> 팀 예산 관리나 작업 효율 점검 시 활용하세요.

## 컨텍스트 관리

### 📌 /context

- **역할**: 현재 컨텍스트 창의 사용 현황을 시각적으로 확인
- **컨텍스트란?**
  - 현재 세션에서 Claude가 기억하고 있는 모든 대화 내용과 파일 정보
  - 모델마다 최대 컨텍스트 크기가 다름 (Sonnet 기준 약 200K 토큰)
- **컨텍스트 부족 시 발생하는 문제**:
  - 오래된 대화 내용이 자동으로 잘려나감
  - Claude가 앞서 논의한 내용을 기억하지 못할 수 있음

```bash
/context
```

> **실무 팁**: 컨텍스트가 70% 이상 차면 `/compact`로 수동 압축을 권장합니다. 자동 압축보다 중요한 정보가 더 잘 보존됩니다.

---

### 📌 /compact

- **역할**: 대화 내용을 요약(압축)하여 컨텍스트 공간을 확보
- **동작**:
  - 지금까지의 대화를 핵심 정보 위주로 요약
  - 불필요한 반복 내용을 제거해 토큰 절약
  - 압축 방식에 대한 지시를 직접 줄 수 있음

```bash
/compact                            # 기본 압축
/compact 코드 변경사항 위주로 요약해 줘  # 특정 내용 중심으로 압축
/compact 현재 버그 수정 진행상황만 남겨줘 # 진행중인 작업 위주로 압축
```

> 장시간 작업 세션에서 특히 유용합니다. 압축 후에도 Claude는 핵심 작업 흐름은 기억합니다.

---

### 📌 /add-dir

- **역할**: 현재 컨텍스트에 추가 디렉토리를 포함
- **동작**:
  - 지정한 폴더의 파일 구조와 내용을 Claude가 참조할 수 있게 함
  - 모노레포처럼 여러 패키지가 분리된 프로젝트에서 유용

```bash
/add-dir ../shared-utils           # 상위 폴더의 다른 패키지 추가
/add-dir ./packages/core           # 하위 패키지 추가
```

> 기본적으로 Claude는 현재 작업 디렉토리만 인식합니다. 다른 폴더의 코드를 참조해야 할 때 사용하세요.

---

### 📌 /memory

- **역할**: 세션을 넘어 영구적으로 유지되는 Claude의 메모리를 관리
- **메모리란?**
  - 대화가 끝나도 사라지지 않고 다음 세션에서도 자동으로 참조되는 정보
  - 사용자 정보, 작업 피드백, 프로젝트 컨텍스트, 외부 리소스 위치 등을 저장
  - 프로젝트 관련 정보를 확인하고 관리하며 기억하고 있는 내용을 확인하고 필요시 수정하거나 삭제할 수 있음
- **메모리 유형**:
  - `user` — 사용자의 역할, 선호도, 기술 수준 등 개인 프로필
  - `feedback` — Claude의 작업 방식에 대한 지침 (피해야 할 것, 유지할 것)
  - `project` — 현재 진행 중인 작업, 목표, 의사결정 배경 등
  - `reference` — 외부 시스템의 위치 정보 (Linear 프로젝트, Grafana 대시보드 등)
- **동작**:
  - 저장된 메모리 목록 및 내용 확인
  - 메모리 추가 / 수정 / 삭제

```bash
/memory                            # 저장된 메모리 목록 확인
```

> 메모리를 활용하면 Claude가 매번 설명하지 않아도 사용자의 선호도나 프로젝트 맥락을 자동으로 파악합니다. 대화 중 "~을 기억해줘"라고 말하면 Claude가 적절한 유형의 메모리 파일에 저장합니다.

## 설정 및 환경

### 📌 /config

- **역할**: Claude Code의 동작 설정을 확인하거나 변경
- **설정 가능한 항목**:
  - 응답 언어 / 출력 형식
  - 파일 저장 및 편집 권한
  - 자동 실행 허용 범위
  - 모델별 기본값

```bash
/config                            # 현재 설정 목록 확인
/config set output.language ko    # 응답 언어를 한국어로 설정
```

> 프로젝트마다 다른 설정을 적용하려면 프로젝트 루트의 `.claude/settings.json`을 직접 편집하는 것도 가능합니다.

---

### 📌 /doctor

- **역할**: Claude Code 실행 환경을 점검하고 문제를 진단
- **점검 항목**:
  - API 키 설정 여부 및 유효성
  - Node.js 버전 호환성
  - 필수 의존성 설치 상태
  - 네트워크 연결 상태
  - 설정 파일 오류 여부

```bash
/doctor
```

> Claude Code가 예상대로 동작하지 않을 때 가장 먼저 실행해보세요. 대부분의 환경 문제를 빠르게 파악할 수 있습니다.

---

### 📌 /permissions

- **역할**: Claude Code가 수행할 수 있는 작업의 권한을 관리
- **확인 및 설정 가능한 권한**:
  - 파일 읽기 / 쓰기 / 삭제
  - 터미널 명령어 실행
  - 네트워크 요청
  - Git 작업 (commit, push 등)

```bash
/permissions                       # 현재 권한 목록 확인
/permissions allow bash            # 터미널 명령어 실행 허용
/permissions deny file.delete      # 파일 삭제 권한 거부
```

> 보안이 중요한 환경에서는 최소 권한만 허용하고, 필요할 때 추가하는 방식을 권장합니다.

---

### 📌 /mcp

- **역할**: MCP(Model Context Protocol) 서버를 관리
- **MCP란?**
  - Claude가 외부 도구나 서비스와 연결할 수 있게 해주는 프로토콜
  - 예: GitHub, Jira, Slack, 데이터베이스 등과 Claude 연동
- **동작**:
  - 연결된 MCP 서버 목록 확인
  - 서버 추가 / 제거
  - 연결 상태 점검

```bash
/mcp                               # 연결된 MCP 서버 목록 확인
/mcp add github                    # GitHub MCP 서버 추가
```

> MCP를 활용하면 Claude가 GitHub PR 조회, Jira 티켓 참조 등을 직접 수행할 수 있어 개발 워크플로우가 크게 향상됩니다.

---

### 📌 /agents

- **역할**: 사용 가능한 서브에이전트(subagent) 목록을 확인하고 관리
- **에이전트란?**
  - 특정 역할에 특화된 AI 서브에이전트로, 복잡한 작업을 병렬 또는 순차적으로 수행
  - 예: 코드 탐색 전문 에이전트, 코드 리뷰 에이전트, 보안 점검 에이전트 등
- **동작**:
  - 등록된 에이전트 목록 및 설명 확인
  - 에이전트별 사용 가능한 도구(Tool) 범위 파악
  - 에이전트 추가 / 제거

```bash
/agents                            # 사용 가능한 에이전트 목록 확인
```

> 대규모 리팩토링이나 멀티스텝 작업 시 적합한 에이전트를 선택하면 작업 효율과 정확도가 높아집니다. `/mcp`로 외부 도구를 연결하고 에이전트에 할당하는 방식으로 확장할 수 있습니다.

## 작업 및 출력

### 📌 /export

- **역할**: 현재 작업 세션과 프로젝트를 파일로 내보내기
- **내보낼 수 있는 내용**:
  - 현재 대화 내용 전체
  - 변경된 소스코드
  - 작업 요약 및 수정 기록
- **활용 상황**:
  - 작업 환경을 다른 PC로 옮길 때
  - 팀원에게 작업 내용을 공유할 때
  - 작업 기록을 아카이브할 때

```bash
/export                            # 기본 내보내기
/export --format markdown          # 마크다운 형식으로 내보내기
```

---

### 📌 /review

- **역할**: 현재 브랜치의 변경사항에 대해 코드 리뷰 수행
- **동작**:
  - Git diff를 분석하여 변경된 코드를 검토
  - 버그 가능성, 코드 품질, 보안 이슈, 개선점 등을 리포트
  - PR 제출 전 사전 점검 용도로 활용

```bash
/review                            # 현재 브랜치 전체 변경사항 리뷰
/review src/api/users.ts           # 특정 파일만 리뷰
```

> PR을 올리기 전에 `/review`를 실행하면 놓친 버그나 리뷰어가 지적할 사항을 미리 파악할 수 있습니다.

---

### 📌 /vim

- **역할**: 입력 모드를 Vim 키바인딩으로 전환
- **동작**:
  - `i` — 입력 모드 진입
  - `Esc` — 명령 모드 전환
  - `j/k` — 커서 이동
  - 일반적인 Vim 단축키 대부분 지원

```bash
/vim                               # Vim 모드 활성화
/vim off                           # Vim 모드 비활성화
```

> Vim에 익숙한 개발자라면 터미널 입력 효율을 높일 수 있습니다.

---

### 📌 /help

- **역할**: 사용 가능한 명령어 목록과 사용법을 출력
- **동작**:
  - 모든 슬래시 명령어 목록 표시
  - 특정 명령어의 상세 설명 출력

```bash
/help                              # 전체 명령어 목록
/help compact                      # /compact 명령어 상세 설명
```

> 명령어가 기억나지 않을 때 언제든지 참고하세요.

---

### 📌 /bug

- **역할**: Claude Code 자체의 버그나 문제를 Anthropic에 보고
- **동작**:
  - 현재 세션 정보와 오류 내용을 자동으로 수집
  - GitHub Issues 페이지로 연결하거나 직접 리포트 제출

```bash
/bug
```

> Claude Code가 이상하게 동작하거나 예상치 못한 오류가 발생할 때 사용하세요.

## 빠른 참조표

| 명령어 | 목적 | 자주 쓰는 시점 |
|--------|------|---------------|
| `/init` | 프로젝트 초기화 | 프로젝트 처음 시작 시 |
| `/status` | 세션 상태 확인 | 작업 중 수시로 |
| `/model` | 모델 변경 | 복잡도에 따라 전환 |
| `/resume` | 이전 세션 복구 | 작업 재개 시 |
| `/clear` | 세션 초기화 | 새 주제 시작 시 |
| `/cost` | 비용 확인 | 비용 관리 시 |
| `/context` | 컨텍스트 확인 | 작업이 길어질 때 |
